# Projects

*Active build work. One folder per project.*

Each project gets its own folder: `Projects/<kebab-case-name>/`, with at
minimum a `README.md` stating the goal, current status, and links to any
`Systems/`/`Reference/` material it depends on.

**Rule:** if you're not actively executing on it, it doesn't belong here.
Paused for more than a few weeks with no movement → move it to
`Archive/` (`git mv Projects/<name> Archive/<name>`). Revive it later by
moving it back.

Does not belong here: reusable patterns or decisions that outlive this
one project — those go in `Systems/` and get linked from here instead of
duplicated.
