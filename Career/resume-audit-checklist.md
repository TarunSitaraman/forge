# Resume Audit Checklist

*Guidance: run this against your actual resume file before sending it
anywhere — not a general "what makes a good resume" essay. Check each
line as pass/fail with the specific fix if failed.*

---

## Per-bullet check

For every bullet on the resume:
- [ ] Leads with outcome/impact, not just activity ("reduced X by Y%,"
      not "responsible for X")
- [ ] Has a real, defensible number if one plausibly exists — if not,
      flagged for follow-up, not left generic
- [ ] Uses a strong verb, not a weak opener ("led," "built," "cut" —
      not "helped with," "worked on")
- [ ] Fits on one line at normal font size — if not, cut it down
- [ ] Would survive you being asked "walk me through this" in an
      interview, in specific detail

*(Run `Technologies/Prompt-Library/Career/resume-bullet-impact-rewrite.md`
against any bullet that fails.)*

## Whole-document check

- [ ] One page (unless 8+ years of relevant experience)
- [ ] No typos, no inconsistent date formats, no inconsistent tense
      (past roles = past tense, current role = present tense)
- [ ] Most relevant/impressive experience is above the fold — don't bury
      it under a long objective statement or oldest-first ordering that
      hides recent work
- [ ] Skills section lists only skills you can defend in an interview —
      no keyword-stuffing you can't back up
- [ ] Tailored for the specific role category being applied to (see
      `Technologies/Playbooks/internship-applications.md` for the tailoring
      step) — not a single generic version sent everywhere
- [ ] File name is professional (`firstname-lastname-resume.pdf`, not
      `resume_final_v3(2).pdf`)
- [ ] Exported as PDF unless the application system specifically wants
      another format

## Before sending, ask yourself

- [ ] If a hiring manager has 6 seconds, what's the first thing they see
      — is it your strongest, most relevant point?
- [ ] Does this resume make the case for THIS specific role, or could it
      be sent to any job unchanged?
