# Technologies

*Reusable technical systems, reference materials, and operational procedures.*

Technologies is where all production-grade technical material lives — the prompts, templates, playbooks, documentation, and project scaffolding you reuse across projects to execute faster.

## What belongs here

- **Prompt-Library/** — 79 operating-procedure-grade prompts across 28 categories. When to use: any decision where structured thinking speeds up a correct answer.
- **Docs/** — Authoritative reference manuals, one per technology (Azure, Databricks, Docker, Git, LangChain, RAG, Vector Databases, LLMs, Prompt Engineering, Markdown). When to use: you need the canonical explanation of how something works.
- **Templates/** — 21 production-quality, reusable document templates (feature specs, deployment checklists, risk registers, retrospectives, decision logs). When to use: starting a standard deliverable.
- **Playbooks/** — 18 execution workflows for recurring operations (deployment, repository cleanup, prompt creation, etc.). When to use: following a tested procedure for a common task.
- **Project-System/** — The one engineering-notebook template every new project scaffolds from. When to use: starting a new project.

## What doesn't belong here

- Notes about a specific project (those belong in Projects/).
- Learning materials or courses (those belong in Courses/).
- Career development resources (those belong in Career/).
- External resources and recommendations (those belong in Resources/).
- Personal technical reference notes outside these systems (those belong in Reference/).

## How to use this folder

All content here is production-ready and meant to be reused. Navigate by category within each subfolder — the Prompt-Library has 28 categories, Templates are organized by document type, Playbooks by task. Every file should feel like a system you lean on repeatedly, not one-off reference material.
