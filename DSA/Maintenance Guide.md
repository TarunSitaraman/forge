# Maintenance Guide

## Naming Rules
Use readable Title Case names and stable canonical filenames.

## Folder Rules
Place each page in its canonical folder. Do not create new DSA folders without updating the architecture.

## Canonical Source Rules
One concept, one home. Other pages link to it.

## Review Checklist
- [ ] Metadata is complete
- [ ] Links resolve in Obsidian
- [ ] No duplicate concept page exists
- [ ] Page follows its template
- [ ] Related index is updated

## Update Workflow
Search first, edit canonical pages second, update indexes third, validate last.

## Archive Strategy
Move deprecated pages to the global archive only after replacement links are in place.

## Broken Link Prevention
Use exact wiki-link page names and run periodic link review.

## Dataview Considerations
Keep types, tags, and status values consistent.

## Future Scalability
Favor small canonical pages connected by links over large duplicated guides.
