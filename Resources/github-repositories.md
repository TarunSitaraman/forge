# GitHub Repositories

*Repos worth studying — reading the code and the issue/PR history to
learn how experienced engineers actually solve problems — not just repos
you depend on as a library. A repo earns a place here for what it
teaches, not for its star count.*

---

## Admission criteria

Add a repo only when you can state which of these it satisfies:
- **Exemplary engineering practice** — code you'd point a junior engineer
  to as "this is how to structure X."
- **Genuinely instructive history** — issues/PRs/commit history that show
  real design debates and tradeoffs, not just a finished artifact.
- **A reference implementation** of a pattern/algorithm you want to
  understand deeply by reading real code, not a toy example.
