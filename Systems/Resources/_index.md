# Resources

*A curated list, not a link dump. Every entry states WHY it earns a
place — the specific thing it does better than alternatives, or the
specific gap it fills. If an entry can't pass that bar, it doesn't
belong here, no matter how popular it is.*

## The admission rule

Before adding anything to any page in this module, answer: **"why this,
specifically, over the ten other things covering the same ground?"** If
the honest answer is "it's popular" or "someone recommended it," it's not
ready to be curated yet — use it first, then add it once you can state
the specific, concrete reason.

## A note on how this module was seeded

The categories below (Official Documentation, GitHub Repositories,
YouTube Channels, Courses, Blogs, Cheatsheets, Newsletters, Podcasts,
Research Papers) exist as structure from day one, but most are seeded
empty or near-empty rather than pre-filled with generic "best of"
recommendations. Channels, courses, and newsletters change hands, go
stale, or get discontinued — a curated list is only trustworthy if every
entry reflects something *you've* actually verified is currently good.
Fill each row in as you personally validate a resource, with the real
reason it earned its place.

The exception is **Official Documentation**: primary sources are stable
by nature (they're maintained by the tool's own creators) and are safe to
seed directly — see `official-documentation.md`.

## Relationship to `Systems/Docs/`

`Systems/Docs/` is Forge's own authoritative reference, written and
owned here. `Systems/Resources/` points outward — to other people's
documentation, teaching, and writing. Don't duplicate `Systems/Docs/`
content into a Resources page; link to the official docs instead and let
`Systems/Docs/` be the synthesized version.

## Maintaining this module

Review each page every few months: does every entry still deserve its
place? A resource that was excellent two years ago may be stale,
superseded, or discontinued now. A curated list that's never pruned
slowly turns into the giant list this module explicitly exists to avoid.
