# Templates

*Production-quality, reusable Markdown templates. Every one includes
inline placeholders, guidance comments on how to fill them in, and a
"Best practices" section at the bottom — copy the file, fill it in,
delete the guidance you don't need.*

## How to use a template

1. Copy the file into the project/location it belongs (most go in a
   `Projects/<name>/` folder; a few — like Weekly Review — are personal
   and can live wherever you track them).
2. Fill in the placeholders. Delete the *guidance* italic text and "Best
   practices" section once the document is real — those are scaffolding,
   not part of the final artifact.
3. Delete any section that doesn't apply. A template is a starting
   structure, not a checklist every section must satisfy.

## Adding a template

Only add one you've needed more than once. A single-use document
structure belongs in the project it was written for, not here.
